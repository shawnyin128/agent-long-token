Multi-Agent Debate — Claim-Level Gain Attribution & Compression

CSCI3033 LLM Reasoner · Final Project · Plan Overview · 2026-04-21

Spec Approved GSM8K · 100 questions 3 agents × 3 rounds Qwen2.5-7B-Instruct (vLLM) 3-day timeline pure-code architecture

Research Question

On GSM8K, which claims exchanged during multi-agent debate causally contribute to the accuracy gain, and can a data-supported claim-level compression rule reduce token cost while preserving accuracy?

Pivot from original MMLU proposal per professor feedback: first pick a task with robust debate gains, then attribute the gains, then design compression.

Key Methodological Stance

No hand-weighted composite importance score. Analysis phase produces only: (a) 4 independent per-claim signals — flip_coincidence, novelty, referenced_later, position; (b) causal Δ_t per claim type via type-level ablation. Compression policy on Day 3 is the simplest rule supported by this evidence. Null result is an acceptable outcome.

3-Day Timeline

Day 1 — Infrastructure + Pilot + Collection (~12h)

AM (4h): Package layout · config · llm_client + cache · dataset loader · answer parser · smoke tests

Midday (1h): Pilot on 30 questions, single-agent vs 3×3 debate

⚠ Gate 1: if acc_debate − acc_single < 3pp → switch to API model (GPT-4o-mini, ~$2–5) and re-pilot. Still <3pp → escalate to user.

PM (5–6h): Full 100-question collection (background) · finalize claim extraction prompt + schema

Day 2 — Extraction + Attribution (~12h)

AM (3h): Run schema-guided claim extraction · 10-dialogue manual spot check

Midday (3h): Flip-point localization · compute 4 independent signals per claim

PM (4–6h): Type-level ablation on ~20 sampled single_wrong ∧ debate_right dialogues · clustering (optional)

⚠ Gate 2: if all Δ_t within noise and no signal correlates with flip path → accept null result; Day 3 reframes as descriptive comparison.

Day 3 — Compression + Evaluation + Report (~12h)

AM (3h): Read Day 2 findings → write policy.json (simplest data-supported rule) → implement compress.apply

Midday (3–4h): Evaluation sweep: 5 methods × 100 questions × {acc, tokens, acc_per_1k}

PM (4–5h): Pareto curve · LaTeX tables · IMRAD report ~6–8 pages · export PDF

Claim Taxonomy (6 types)

proposal

introduces new candidate answer or reasoning path

evidence

supports a position with computation / derivation

correction

identifies error in prior claim, offers fix

agreement

affirms prior claim, no new info

question

directed at another agent

other

pleasantries, meta-discourse, off-topic

Top Divergence Risks

high
Claim extraction JSON malformed — corrupts signals → flip analysis → ablation → policy. Mitigation: retry + extraction_failed flag + Day 2 AM spot check.
high
Answer parser ambiguity — "$42" vs "42" etc. corrupts accuracy → gate decisions. Mitigation: smoke-tested variants + unparseable outputs logged for manual inspection.
med
Debate turn stochasticity (temp=0 but GPU batch effects). Mitigation: single-run acknowledged in limitations.
med
Ablation replay divergence. Same mitigation as above.
med
Cache corruption on crash. Mitigation: startup validates last line, atomic appends.
