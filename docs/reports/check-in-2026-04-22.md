# 1. Response to Proposal Feedback

The original proposal planned a bottom-up structural analysis of
multi-agent debate (MAD) on MMLU. The instructor flagged two risks:

1. **Debate on MMLU is unreliable.** MAD-style debate often fails to
   beat single-agent baselines on MMLU, so a structural analysis
   would be dissecting noise rather than a real phenomenon.
2. **Gain must exist before attribution makes sense.** The project
   should first verify that debate produces a robust gain on some
   task, then analyze *where* that gain comes from, and only then
   design a compression rule around the gain-bearing structure.

I adopted this feedback in full by pivoting the design (see
`docs/design-docs/2026-04-21-multi-agent-debate-claim-analysis-design.md`):

- **Task switched from MMLU → GSM8K**, where MAD has documented,
  reproducible gains in the literature (Du et al. 2023).
- **Analysis reordered into a gate-based workflow**: Gate 1 verifies
  debate gain (≥ 3 pp), Gate 2 measures per-claim-type causal
  effect, Day 3 designs compression based on findings.
- **Spec §5.4 explicitly preserves a null-result path**: if
  attribution finds no signal, Day 3 becomes a descriptive
  comparison rather than a data-supported rule.

# 2. Related Work

**Multi-agent debate.** Du et al. (2023, *Improving Factuality and
Reasoning in Language Models through Multiagent Debate*) showed that
3–5 agents iteratively critiquing each other's answers outperform
single-agent on multiple QA benchmarks, including GSM8K. This paper
is the direct reference for the 3×3 (3 agents × 3 rounds) protocol
used here.

**Chain-of-thought + self-consistency.** Wei et al. (2022) and Wang
et al. (2022, *Self-Consistency Improves Chain of Thought Reasoning*)
established that sampling multiple reasoning chains and majority-
voting over final answers outperforms a single chain — a parallel
mechanism to what debate claims to provide. The central question of
this project, as the preliminary results will show, is whether MAD
is meaningfully different from self-consistency once you strip the
dialogue layer.

**Prompt compression.** LLMLingua (Jiang et al. 2023) compresses
prompts by removing low-perplexity tokens; sliding-window and
uniform random-drop baselines follow established practice. My
compression experiments will reuse these baselines (`b3` sliding
window, `b5` seeded random drop) as references for the `ours`
data-driven rule.

**Claim-level attribution in dialogue.** I am not aware of prior
work that formalizes debate messages as typed claims and runs
span-level ablation; the taxonomy (proposal, evidence, correction,
agreement, question, other) in this project is bespoke for MAD
transcripts (spec §3.4).

# 3. Method

**Dataset.** GSM8K test split, 100 questions, seed 42.

**Model.** Qwen2.5-7B-Instruct served via vLLM on an NYU Greene A100
node, OpenAI-compatible endpoint. Temperature 0.

**Debate protocol.** 3 agents × 3 rounds. Agents use differentiated
system prompts (solver, skeptic, synthesizer) from the MAD literature
pattern. Final answer is a majority vote over the last round's
per-agent parsed answers.

**Claim extraction.** Per-message schema-guided extraction via a
6-type taxonomy (proposal, evidence, correction, agreement, question,
other), with pydantic validation and one stricter-prompt retry on
schema failure. Each claim carries a verbatim
`source_message_span` pointer back into the original message text.

**Gate 2 — type-level ablation.** For each of 6 claim types *t*,
blank every message (rounds 1…N-1) that contains any claim of type
*t* in the `single_wrong ∧ debate_right` subset (the canonical
"debate flipped a wrong answer correct" subset), replay the final
round through the 3 agents, recompute accuracy, and report
Δ_t = acc(with) − acc(without). Coverage constraint: message-level
masking was required because span-level coverage averaged only 26 %
of each message and produced mechanically zero signal.

**Control experiment (added in response to ablation null result).**
Same subset, blank *all* messages in rounds 1…N-1 (no type filter —
only the question and debate structure remain), replay and measure
Δ_control = acc(with) − acc(without_all). If Δ_control ≈ 0 the null
result is real; if Δ_control > 0 the single-type signal was simply
drowned by the LLM's ability to re-solve from the question alone.

# 4. Preliminary Results

All preliminary artifacts are under `artifacts/` in the repo; see
`artifacts/pilot_report.md`, `artifacts/analysis/gate2_report.md`,
and `artifacts/analysis/control_result.json` for the raw numbers.

## 4.1 Gate 1 — debate *does* help on GSM8K

100 questions, single-agent vs 3×3 debate:

| method | n | correct | accuracy |
|---|---|---|---|
| single | 100 | 87 | 86.7 % |
| debate (3×3) | 100 | 93 | 93.3 % |

**Δ = +6.7 pp**, well above the 3 pp threshold, so Gate 1 passes and
the pipeline continues. This replicates the MAD-literature range of
3–10 pp on GSM8K.

*Note on discovered issues (included because they changed my
interpretation of prior runs).* Early Gate 1 runs reported
Δ = 0 pp, which looked like a soft fail. Two parser bugs caused it:
(a) Qwen learned the literal `N` from the prompt instruction
"end with `#### N` where N is your final numeric answer" and
produced `#### N66`; my regex required an immediate digit after
`####` and fell back to a first-`$<num>` heuristic that picked up
`$16` (a book price) instead of the correct total `66`. (b) Claim
JSON payloads contained LaTeX (`\(`, `\frac`) which broke
`json.loads` via invalid-escape errors. Both are fixed in the code
and documented in the design doc; the accuracy numbers above are
from the post-fix run. This was a useful reminder that
"preliminary results" include verifying that the measurement
instrument itself is valid.

## 4.2 Claim taxonomy yields a heavily skewed distribution

Across 2 846 extracted claims from 100 dialogues:

| type | count | % |
|---|---:|---:|
| proposal | 1 249 | 43.9 % |
| evidence | 1 114 | 39.1 % |
| agreement | 430 | 15.1 % |
| other | 44 | 1.5 % |
| correction | 9 | 0.3 % |
| question | 0 | 0.0 % |

Three categories account for 98 % of claims. A keyword sanity check
on 20 skeptic round-2/3 messages found only 1/20 containing
error/wrong/mistake terminology — and that one was "all steps appear
to be correct", not an actual correction. Interpretation: **on
GSM8K 3×3 debate with Qwen2.5-7B, agents behave cooperatively, not
adversarially**. There is very little to correct and nothing to ask
about once three agents have computed a short arithmetic chain.

A 10-dialogue spot check (AI-assisted with a 6-item checklist,
results in `artifacts/spot_check.csv`) produced 10 / 10 dialogues
passing at mean 5.6 / 6, confirming extraction quality is adequate
for downstream attribution.

## 4.3 Gate 2 — type-level ablation returns null

On the *single_wrong ∧ debate_right* subset (n = 9), for every one
of the 6 claim types:

| type | n | Δ | class |
|---|---:|---:|---|
| proposal | 9 | +0.000 | noise |
| evidence | 9 | +0.000 | noise |
| correction | 9 | +0.000 | noise |
| agreement | 9 | +0.000 | noise |
| question | 9 | +0.000 | noise |
| other | 9 | +0.000 | noise |

Every Δ is exactly zero. `make gate2` returns exit code 10 (null
result) per the thresholds defined in `cli/ablate.py` (LIKELY
|Δ| ≥ 0.10, NOISE |Δ| ≤ 0.03, PASS on at least one type |Δ| ≥ 0.05).

## 4.4 Control experiment — null is real, not an ablation artifact

The natural skeptical reading of section 4.3 is that single-type
ablation is simply too weak: if an agent can re-solve a GSM8K
problem independently from the question, deleting one claim type
will not hurt. To rule this out I ran a control: blank *all*
messages in rounds 1…N-1 on the same 9-qid subset and replay.

| condition | n | acc(with) | acc(without) | Δ |
|---|---:|---:|---:|---:|
| drop single type (avg) | 9 | 1.000 | 1.000 | +0.000 |
| **drop ALL messages**  | 9 | 1.000 | 1.000 | **+0.000** |

Even when the entire debate history is replaced with empty strings,
leaving only the question and the turn-structure (`agent 0 round 1`,
`agent 0 round 2`, …), all 9 subset questions are still answered
correctly by the three-agent majority vote. A sanity dump of one
masked transcript confirms that the mask is real: five of six
rounds-1-and-2 messages are blanked to the empty string.

## 4.5 Interpretation

Taken together:

> The +6.7 pp debate gain on GSM8K with Qwen2.5-7B **cannot be
> attributed to information exchanged between agents**. It emerges
> from majority voting over three independent re-solves. "Debate"
> in this regime functions as ensembling / self-consistency with
> extra steps, not as a dialogue that transfers information.

This is consistent with the sparse-type distribution in §4.2 (there
is almost no *correction* or *question* content for information
transfer to be conveyed in) and is the spec §5.4 null-result path.

# 5. Limitations & Planned Next Steps

The preliminary results deliberately stop here. Before running the
compression sweep (Day 3 of the original plan) I want to verify two
confounders that could invalidate the "debate is voting" thesis:

1. **Model specificity.** All results are on a single model
   (Qwen2.5-7B-Instruct). Qwen's single-agent GSM8K accuracy is
   already 86.7 %, so its residual error rate is only 13.3 %, which
   might make the message content feel unimportant even if it would
   matter at a lower baseline. I will replicate Gate 1 + Gate 2 +
   control on at least one other model (gpt-4o-mini or
   Llama-3.1-8B-Instruct) and report whether the null replicates.
   If not, "voting, not dialogue" is a Qwen observation, not a
   task-level property.

2. **Debate prompt design.** The current solver/skeptic/synthesizer
   prompts are standard MAD-style but may simply be too cooperative
   for Qwen on easy arithmetic. A stronger variant (skeptic: "you
   **must** find at least one disagreement with another agent's
   reasoning"; synthesizer: "ask one clarifying question before you
   answer") could change the claim distribution (more *correction*,
   more *question*) and potentially restore ablation signal. If
   aggressive prompts still produce Δ = 0, the voting hypothesis is
   strengthened; if they do not, the original prompts were
   under-driving adversarial behavior.

Both experiments reuse the existing LLM cache for the questions that
overlap, so the marginal cost is modest.

**After those two confounders are ruled out**, the remaining work is
Day 3 of the original plan:

- Define the `ours` compression rule (heuristic given null Gate 2:
  drop *agreement* and *other* claims as lowest-information-density
  classes).
- Run `make evaluate` — 5 methods (b1 full, b2 single, b3
  last-k-rounds, b5 random-drop, ours) × 100 questions × (accuracy,
  total tokens, accuracy-per-1k-tokens).
- Produce the Pareto figure comparing the 5 methods and the IMRAD
  report.

The predicted Pareto under the voting-not-dialogue thesis: b1, b3,
b5, and ours all plateau near 93 % accuracy at widely different
token counts, with b2 dropping to ≈ 87 % because it removes the
voting structure. If that prediction is verified, the paper's
central figure and main claim are both supported; if ours does not
materially beat b5, the "data-supported rule" cannot outperform
random drop — itself a legitimate negative finding.

# 6. Code, Reproducibility, and Artifacts

The project ships as a Python package `agentdiet` (≈ 3 000 LoC, 35
CLI-backed `make` targets, 400 + unit + smoke tests) under MIT. All
experiments in this check-in are reproducible from a fresh clone via
the Makefile runbook documented in `README.md`:

```
make serve && make health          # start vLLM
make pilot && make gate             # Gate 1 — PASS +6.7pp
make collect && make collect-report # 100/100 ok
make extract && make extract-report # 94 ok, 6 partial, 0 failed
make spot-check                     # 10-dialogue review
make analyze                        # flip events + signals
make ablate && make gate2           # Gate 2 — NULL
make ablate-control                 # control — Δ=0 confirms null
```

Every feature in `.claude/features.json` is associated with a
per-feature Planner → Generator → Evaluator YAML in
`.claude/agents/state/archive/*/`, and each fix (parser, LaTeX
escapes, message-level ablation, drop-all control) has its own
commit with a detailed message. Code review was multi-stage — AI
reviewer per feature plus a post-hoc sp-feedback Mode A over all
features for systematic issues.
